# 404 Not Found Page — Design Spec

**Date:** 2026-04-28  
**Status:** Approved

---

## Goal

Add a branded, localized 404 page that matches the Math on Canvas landing page visual identity. Currently the project shows Next.js's default 404 page with no branding or navigation.

---

## Architecture

### File placement

```
src/app/not-found.tsx          ← root-level 404 handler (catches all unmatched routes + notFound() calls)
src/app/NotFoundPage.css       ← scoped styles for the 404 page
```

No `app/[lang]/not-found.tsx` is needed: the root-level file covers both invalid lang codes (`/fr`) and any unmatched routes (`/anything`).

### Language detection (fallback chain)

The `not-found.tsx` is a Next.js Server Component and has no access to route params. Language is resolved in this order:

1. **URL path** — parse the first segment of the path from the `next-url` header (e.g. `/ru/foo` → `ru`). Validate against `LANGUAGES`.
2. **Accept-Language header** — pass the first value to `normalizeLanguage()` from `src/lib/i18n/constants.ts`.
3. **Default** — `'en'`.

All header access uses `headers()` from `next/headers` (async Server Component).

## SEO

Next.js automatically sends HTTP 404 status when `not-found.tsx` renders — no manual action needed.

Export a `metadata` object with:

```ts
export const metadata: Metadata = {
  title: '404 — Math on Canvas',   // localized per lang in generateMetadata
  robots: { index: false, follow: false },
}
```

Because language is determined at runtime (from headers), `generateMetadata` cannot be used (it requires static params). Instead, set a **static** `metadata` export with a generic title and `noindex`. Localized `<title>` will be rendered via a `<title>` tag inside the component itself using a `<Head>`-equivalent pattern — or simply accepted as a trade-off (generic title for 404 is acceptable for SEO since the page is `noindex` anyway).

**Decision:** Use static `metadata` with `robots: noindex`. The `<h1>` on the page IS localized (good for users), the `<title>` stays generic (acceptable since noindex).

## Translations

Add a `notFound` namespace to each locale file. Since `not-found.tsx` is a Server Component that directly imports JSON (same pattern as `LandingPage.tsx`), no i18next runtime is used.

New files: `src/lib/i18n/locales/{lang}/not-found.json` for all 4 languages.

English template:

```json
{
  "heading": "Page Not Found",
  "description": "This page has been moved or no longer exists. Try going back to the home page or opening the editor.",
  "goHome": "Back to Home",
  "openEditor": "Open Editor",
  "footer": "© 2025 Math on Canvas"
}
```

**Import pattern** (same as `LandingPage.tsx` — direct static imports, no i18next runtime):

```ts
import notFoundEN from '@/lib/i18n/locales/en/not-found.json'
import notFoundRU from '@/lib/i18n/locales/ru/not-found.json'
import notFoundES from '@/lib/i18n/locales/es/not-found.json'
import notFoundDE from '@/lib/i18n/locales/de/not-found.json'

const translations = { en: notFoundEN, ru: notFoundRU, es: notFoundES, de: notFoundDE }
const t = translations[lang] ?? translations.en
```

## Component Structure

`not-found.tsx` is a single async Server Component — no sub-components needed.

```
not-found.tsx
  ├── headers() call → lang detection
  ├── import not-found JSON for detected lang
  ├── <div.notfound>          ← grid texture wrapper (same as .landing)
  │   ├── <header>            ← brand + goHome link
  │   ├── <main>              ← 404 number + h1 + p + CTA buttons
  │   └── <footer>            ← copyright + version
  └── NotFoundPage.css
```

---

## Out of Scope

- No `app/[lang]/not-found.tsx` — root-level file is sufficient
- No animation or illustration
- No search field or sitemap links
- No dark mode overrides (inherits from `globals.css`)
